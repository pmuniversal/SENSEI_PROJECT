# SIGMA·SCAN — Анализ ниши для микро-SaaS (10 этапов с AI)

> Источник: методология SIGMA от Дмитрий Попов / COMANDOS AI
> Применять: при выборе ниши, запуске нового продукта, оценке рынка.
> Время: 2 дня вместо 6 недель агентской работы (стоимость у агентства: 300К+ ₽)

## Экспресс-тест перед началом (15 минут)

Три вопроса убивают 80% плохих идей:

| # | Вопрос | Где проверить | Красный флаг |
|---|--------|--------------|--------------|
| 1 | Существует западный аналог с платящими клиентами? | Product Hunt, G2, AppSumo | Нет аналога → боль не доказана |
| 2 | Есть живое сообщество где обсуждают эту боль прямо сейчас? | Reddit, Telegram, VC.ru | Тишина → нет активного спроса |
| 3 | Назови 3 человека из ниши которым позвонишь сегодня? | Твои контакты | Никого → ты чужак на рынке |

Все 3 = ДА → запускай полный SIGMA·SCAN. 1 или 0 → смени нишу.

## 10 Этапов SIGMA·SCAN

### Этап 0 — Стартовая точка (30 мин)
Три варианта:
- **А — Работаешь в отрасли** → инвентаризация: что делаешь руками → можно автоматизировать
- **Б — Есть бизнес** → за что платишь фрилансерам? за какой инструмент платил бы ежемесячно?
- **В — Нет идеи** → используй этапы 1-3

**Промт (Claude/ChatGPT):**
```
Я [опиши себя: опыт, профессия]. Предложи 10 потенциальных ниш для микро-SaaS.
Для каждой: боль, кто платит, моё преимущество, сложность входа (1-5).
ТОП-3 для российского рынка.
```

### Этап 1 — Что рынок уже покупает (45 мин)
Источники: **Product Hunt**, **AppSumo**, **G2**, YC Request for Startups.

Промт для Claude in Chrome (открыть producthunt.com):
```
Найди все продукты в категории [НИША]. Для каждого: название, апвоуты,
ТОП-3 просьбы добавить функции, цена. Что адаптировать для России?
```
> Лайфхак: апвоуты = интерес, комментарии "I wish it had..." = список функций MVP.
> AppSumo 500+ покупок = боль реальная.

### Этап 2 — Тренд: растёт или падает (30 мин)
Источники: **Google Trends** (5 лет), **Reddit** ([ниша] software problems).

> Лайфхак: российский рынок отстаёт от западного на 2-3 года.
> Product Hunt 2023 = российский рынок 2025-2026.

### Этап 3 — Адаптация под Россию/СНГ (1-2 часа)
Источники: **Яндекс.Wordstat**, **HeadHunter** (вакансии = скрытые боли), **Авито**.

> Лайфхак HH: зарплата специалиста 80К/мес = 960К/год за ручной труд.
> Твой SaaS за 3К/мес = 100x выгода для клиента.

### Этап 4 — Скоринг: брать или нет (1 час)

Промт:
```
Оцени ниши [ВСТАВЬ 3-5 ГИПОТЕЗ] по 7 критериям (1-5 баллов):
1. Острота боли (платят прямо сейчас?)
2. Размер рынка (достаточно для 1М/год?)
3. Конкуренция (слабые места у лидеров?)
4. Простота MVP (за 1-2 месяца одному?)
5. Повторяющийся платёж (подписка?)
6. Моё преимущество (знаю рынок изнутри?)
7. Западный аналог уже есть?
Таблица + победитель.
```

**Золотая ниша:** западный аналог есть + в России нет/плохой + ты знаешь рынок.

### Этап 5 — Глубокий анализ конкурентов (3-4 часа)
Источники: **G2**, **Capterra**, отзовики (Otzovik.ru, irecommend.ru).

Промт (Claude in Chrome, открыть страницу конкурента):
```
Разбери отзывы на [КОНКУРЕНТ] на три группы:
❌ ЧТО НЕНАВИДЯТ (1-2 звезды) — топ-5 жалоб с цитатами
✓ ЧТО ЛЮБЯТ (4-5 звёзд) — за что терпят минусы
💡 ЧТО ПРОСЯТ ДОБАВИТЬ — готовые идеи MVP
```

> Лайфхак: 1 звезда = золото (что сломано). 3 звезды = бесполезны.

### Этап 6 — Голос клиента (2-3 часа)
Источники: Telegram-чаты ниши, форумы, App Store, 2ГИС, YouTube комментарии.

Промт:
```
Анализируй отзывы [ВСТАВЬ 20+ ОТЗЫВОВ]:
1. ТОП-5 болей языком клиента (с цитатами)
2. Скрытые желания (между строк)
3. Эмоциональные триггеры (повторяющиеся слова)
4. Готовые буллеты для лендинга: "Устали от X? Мы сделали: Y"
5. Killer-feature которую никто не закрывает
```

> Лайфхак: Telegram-чаты богаче любого опроса — люди пишут без цензуры.

### Этап 7 — JTBD анализ (2 часа)

Промт:
```
JTBD-анализ для SaaS в нише [НИША]:
1. КТО ПЛАТИТ vs КТО ИСПОЛЬЗУЕТ (часто разные люди)
2. 3 главные "работы": "Когда я [ситуация], хочу [действие], чтобы [результат]"
3. Барьеры покупки: страхи, сомнения, альтернативы
4. Момент решения: что происходит → "всё, ищу решение"
5. Обещание продукта одним предложением:
   "Для [кто], которые [проблема] — мы [решение], в отличие от [альтернатива]"
```

### Этап 8 — Анализ трафика конкурентов (1 час)
Источники: **SimilarWeb** (бесплатно 5 сайтов/мес), Semrush, SpyFu.

### Этап 9 — Финансовая модель (2 часа)

Промт:
```
Финансовая модель микро-SaaS: ниша [НИША], цена [ЦЕНА]₽/мес, CAC [CAC]₽,
отток 5%/мес, прирост [N] клиентов/мес.
Рассчитай: LTV и LTV/CAC, break-even, MRR по 12 месяцам,
когда 300К/500К/1М ₽/мес. Входить или нет?
```

## 10 Красных флагов — когда закрывать нишу

1. Рынок < 50 000 клиентов в России
2. Средний чек < 500 ₽/мес (1М = 2000+ клиентов — не потянуть одному)
3. Продажи требуют демо + переговоры + договор (нужен self-serve)
4. Жёсткое госрегулирование (медицина, финансы, юриспруденция)
5. 1-2 клиента дадут >30% выручки
6. Конкурент — Google/Яндекс/1С с бесплатным продуктом
7. Ниша сезонная (нет стабильного MRR)
8. MVP требует > 3 месяцев разработки
9. Нет ни одного знакомого из ниши (первые 10 клиентов — из личного окружения)
10. Сам бы не заплатил за это

Правило: 1-2 флага = работаем с рисками. 3+ = меняем нишу.

## 7 Ошибок которые делают 90% новичков

1. Влюбляются в идею и игнорируют данные
2. Слишком широкая ниша ("автоматизация для бизнеса" — не ниша)
3. Оценивают по выручке конкурента, не по боли
4. Пытаются придумать уникальное вместо "скопировать и улучшить"
5. Считают: низкая цена = больше клиентов (290₽ привлекает тех, кто уйдёт)
6. Начинают строить **до** первых 10 предоплат
7. Исследуют слишком долго (2 дня → решение, не месяцы)

> **Правило:** первые 10 предоплат до первой строки кода.

## Кейс: CRM для салонов красоты (пример разбора)

| Показатель | Данные |
|-----------|--------|
| Рынок | 109К салонов в России, 70% без CRM |
| Тренд | +340% запросов за 3 года |
| Конкурент | YCLIENTS (694-3500₽/мес), поддержка 3/10 |
| Боли | Дорого + нет поддержки (38%), no-show (27%), нет мобильного (21%) |
| Unit-экономика | При 990₽/мес: break-even = 142 клиента, LTV/CAC = 24x |
| Вердикт | ВХОДИТЬ |

## Паттерн успешных продуктов

Calendly ($3 млрд), Slack ($27 млрд), amoCRM ($1.5 млрд) — один паттерн:
**Реальная боль + существующая технология + лучший UX или локализация = бизнес.**
