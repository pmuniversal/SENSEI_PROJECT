# Протокол выбора модели Claude

> Подгружается автоматически. Источник: _PM/RULES/model-selection.md.
> Цель: экономить кредиты и лимит чата Kiro — не использовать мощную модель там,
> где справится более лёгкая.

## Главное правило

**Перед каждым следующим шагом** агент называет рекомендуемую модель и причину.
Формат одной строкой в конце ответа:

```
🤖 Следующий шаг: [что делать] → рекомендую [модель] — [причина одной фразой]
```

## Какую модель когда использовать

### Claude Haiku — самая лёгкая (минимум кредитов)
Использовать когда:
- Простые вопросы, уточнения, короткие ответы
- Чтение и резюмирование файлов
- Форматирование, правка текста, перевод
- Повторяющиеся рутинные задачи
- Быстрая проверка «всё ли правильно»

### Claude Sonnet 4.5 — средний уровень (баланс цена/качество)
Использовать когда:
- Написание и правка кода средней сложности
- Составление документов, планов, инструкций
- Анализ данных без глубокой логики
- Большинство рабочих задач в проекте
- Работа с roadmap, чекпоинтами, changelog

### Claude Sonnet 4.6 — текущая рабочая (стандарт по умолчанию)
Использовать когда:
- Архитектурные решения среднего масштаба
- Сложный код с логикой и зависимостями
- Исследование и структурирование новой темы
- Задачи где нужна точность и контекст

### Claude Opus 4 — самая мощная (расходовать осознанно)
Использовать когда:
- Стратегические решения высокого уровня
- Сложная архитектура системы с нуля
- Глубокий анализ бизнеса, рынка, конкурентов
- Задачи где цена ошибки высока
- Нестандартные задачи без готового решения
- Разработка нового продукта / MVP с нуля

---

## Правило экономии

- Не открывать новый чат без необходимости — лимит чата дорогой.
- Группировать мелкие задачи в один запрос вместо многих.
- Если задача решена на Sonnet — не переключаться на Opus.
- Если в середине задачи понял что модель слабовата — сказать владельцу,
  не додумывать самому.

## Пример в работе

```
Задача: написать скрипт парсинга сайта
→ 🤖 Следующий шаг: написать скрипт → рекомендую Sonnet 4.6 — средняя сложность,
  нужна точность логики, Opus избыточен

Задача: придумать архитектуру всей платформы Bekwise
→ 🤖 Следующий шаг: проектировать архитектуру → рекомендую Opus 4 — стратегическое
  решение высокого уровня, цена ошибки высока

Задача: поправить опечатку в README
→ 🤖 Следующий шаг: правка текста → рекомендую Haiku — простая задача,
  Sonnet избыточен
```
