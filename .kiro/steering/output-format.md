# Правило: формат документов для Бекзода

## Обязательное правило (всегда)

Когда создаю документ который Бекзод будет **читать / изучать / давать фидбек** —
создавать его в формате **.docx (Word)**, а не .md.

Word-файлы класть в папку:
`G:\Мой диск\KIRO_PROJECTS\SaaS_UZ_Stores\DOCS\`

Требования к Word-документам:
- Заголовки (H1, H2, H3) — структурированные
- Таблицы там где есть сравнения, списки с колонками, цифры
- Читаемые абзацы, не стена текста
- Никаких markdown-крючков (# ** `` ---) в содержимом

## Когда .md — допустимо
- Технические файлы кода (README, steering-файлы, чекпоинты для агента)
- Файлы которые читает агент, не человек

## Когда .docx — обязательно
- Продуктовые документы (PRODUCT_OVERVIEW, ROADMAP для бизнеса)
- Коммерческие предложения
- Финансовые модели
- Любой документ где Бекзод сказал «покажи» / «дай почитать»
- Отчёты по итогам этапов
