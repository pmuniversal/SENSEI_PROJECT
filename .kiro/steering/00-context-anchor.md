# 00 — Якорь контекста: Sensei (Telegram-бот)

> Загружается в каждом чате. Не даёт потерять контекст при смене чата/ПК.

## Что это за проект

**Sensei** — Telegram-бот, ассистент владельца. Читает `PROJECT_STATUS.md` всех
проектов портфеля и формирует сводку по команде «ИСП» (Изучи Состояние Проектов).
Уровень 2 в иерархии: _PM (УП) → Sensei-бот → Агенты проектов.

Роль агента: архитектор и инженер бота — помогаешь проектировать, строить и
поддерживать Sensei. Язык — русский. Стек — Python + Telegram Bot API + простое
хранилище (без ORM, без сложностей).

## Где что лежит (карта проекта)

- **Код бота:** в корне `SENSEI_PROJECT/`
- **Правила поведения:** `.kiro/steering/`
- **Статус:** `PROJECT_STATUS.md`
- **История:** `CONTEXT/PROJECT_CHANGELOG.md`
- **Чекпоинт:** `CONTEXT/CHECKPOINT_LATEST.md`
- **Дорожная карта:** `CONTEXT/ROADMAP.md` (создать если нет)

## Связь с системой УП

Sensei читает `PROJECT_STATUS.md` каждого проекта из `KIRO_PROJECTS/`.
Актуальная сводка команды «ИСП» генерируется по этим файлам.
Общие правила поведения — в `.kiro/steering/`.

## Ритуал старта сессии

1. Прочитать этот файл + `CONTEXT/CHECKPOINT_LATEST.md` + `PROJECT_STATUS.md`.
2. Назвать текущий этап (по `CONTEXT/ROADMAP.md`) и ближайший шаг.
3. Ждать задачи от владельца.
