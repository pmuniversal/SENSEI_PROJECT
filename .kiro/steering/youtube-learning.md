# Навык: Изучение YouTube-видео → сводка → внедрение в проект

> Источник: _PM/RULES/youtube-learning.md
> Триггер: владелец кидает ссылку на YouTube-видео.
> Цель: изучить видео → структурная сводка → предложить что внедрить в ТЕКУЩИЙ проект.

## Когда срабатывает

Владелец прислал ссылку вида `youtube.com/watch?v=...` или `youtu.be/...`
(возможно с припиской «изучи», «сделай сводку», «что взять для проекта»).

## Шаг 1 — Получить транскрипцию (по порядку, до первого успеха)

Прямой доступ к YouTube часто блокируется (403/CAPTCHA). Методы по приоритету:

1. **MCP-сервер транскрипции (самый надёжный).** Если установлен YouTube-transcript
   MCP — использовать его. Если нет — предложить владельцу установить (см. Шаг 5 ниже).
2. **web_fetch через reader-прокси:** `https://r.jina.ai/https://youtu.be/{ID}`
   (короткая ссылка без query — проходит фильтр web_fetch). Иногда упирается в CAPTCHA.
3. **web_fetch сервисов транскрипции** (могут отдавать 403): tactiq, youtubetotranscript,
   tubetranscript, notegpt.
4. **Ручной ввод от владельца (гарантированный fallback):** попросить открыть видео →
   под видео «...» → «Показать транскрипцию» → скопировать и вставить в чат.
   Или сервис youtubetotranscript.com → вставить ссылку → скопировать текст.

**ЖЁСТКО (evidence-based):** если транскрипцию получить не удалось — НЕ выдумывать
содержание. Прямо сказать владельцу «не смог получить, вставь транскрипцию вручную».

## Шаг 2 — Профессиональная структурная сводка

После получения транскрипции составить сводку по шаблону:

```
🎬 СВОДКА ВИДЕО: [название]
Источник: [ссылка] | Длительность/объём: [если известно]

## О чём видео (2-3 предложения)
[Суть без воды]

## Ключевые тезисы (по порядку)
1. [тезис] — [краткое пояснение]
2. ...

## Практические приёмы / методики
- [конкретный приём, который можно применить]

## Цитаты/факты, заслуживающие внимания
- [если есть важные цифры, имена инструментов, формулы]

## Что осталось за кадром / требует проверки
- [чего не хватило, что проверить отдельно]
```

Правило: сводка — на основе РЕАЛЬНОЙ транскрипции, без домыслов. Если момент неясен —
пометить `[неясно из транскрипции]`.

## Шаг 3 — Предложения по внедрению в ТЕКУЩИЙ проект

Сводку привязать к проекту, в котором находится агент. Для каждого кандидата:

```
💡 ВНЕДРИТЬ В ПРОЕКТ: [что именно]
   Тип: [навык / опыт / функция / процесс / инструмент]
   Откуда в видео: [тезис №X]
   Зачем этому проекту: [как усиливает — боль/дифференциатор/эффективность]
   Сложность: [низкая/средняя/высокая]
   Quick win: [можно ли быстро попробовать]
   Решение: [внедрить сейчас / в backlog / не подходит]
```

Фильтр релевантности: предлагать ТОЛЬКО то, что реально подходит профилю проекта
(см. 00-context-anchor.md проекта). Не тащить всё подряд.

## Шаг 4 — Зафиксировать

- Полезные методики → записать в `CONTEXT/SKILLS_LOG.md` (через skill-suggestions.md).
- Если внедряется функция → завести в `CONTEXT/BACKLOG.md` или ROADMAP.
- Сводку сохранить в `CONTEXT/LEARNING/[дата]-[краткое-название].md` (создать папку если нет).

## Шаг 5 — Установка MCP-транскрипции (разовая, для надёжности)

Если владелец хочет чтобы это работало автоматически без ручной вставки —
предложить установить YouTube-transcript MCP-сервер (см. skill-search-algorithm.md,
Шаг 1: искать `mcp server youtube transcript` на github.com/punkpeye/awesome-mcp-servers).
Проверить по фильтру (свежесть, звёзды, лицензия) перед установкой.

## Запрещено

- Выдумывать содержание видео, если транскрипция не получена.
- Предлагать к внедрению нерелевантное проекту «потому что в видео было».
- Делать сводку «по названию» — только по реальному содержанию.
