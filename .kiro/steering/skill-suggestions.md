# Протокол предложения навыков агенту

> Подгружается автоматически. Источник: _PM/RULES/skill-suggestions.md.
> Обязателен для всех проектов портфеля.

## Суть правила

Агент проекта **активно изучает проект** и **регулярно предлагает владельцу
навыки (skills/steering-файлы)**, которые сделают работу в проекте лучше, точнее
и быстрее. Навыки ищутся только из **надёжных источников**.

---

## Когда предлагать навыки

1. **При старте нового проекта** — агент читает весь проект (цели, roadmap,
   технологии, контекст) и в конце ритуала старта предлагает 2–5 навыков,
   которые были бы полезны именно здесь.
2. **При появлении новой задачи или направления** — если задача выходит за
   рамки текущих навыков агента, предлагает навык под неё.
3. **На чекпоинте** — раз в значимый этап (когда обновляется ROADMAP.md)
   сверяется: появились ли новые потребности → при наличии предлагает навык.
4. **По запросу владельца** — в любой момент по команде «предложи навыки».

---

## Формат предложения (обязательный)

Всегда предлагать **кратко и по делу**, без давления:

```
💡 Навык: [название]
   Зачем: [одна фраза — что станет лучше/проще]
   Источник: [ссылка или описание: официальная документация / GitHub / проверенное сообщество]
   Тип: [бесплатный / платный — цена]
   Установить? [да / позже / не нужно]
```

Предлагать не более 5 навыков за раз, чтобы не перегружать владельца.

---

## Источники (только надёжные)

Агент ищет и предлагает навыки ТОЛЬКО из:
- Официальная документация инструмента/библиотеки/фреймворка.
- Официальные GitHub-репозитории с высоким числом звёзд и активной поддержкой.
- Проверенные профессиональные сообщества (Stack Overflow, MDN, официальные блоги).
- Известные авторы и курсы, которые уже упоминал владелец (Евгений, Георгий Ривер и др.).

**Запрещено:** предлагать навыки из анонимных источников, малоизвестных блогов,
платных ресурсов без явного указания стоимости, устаревших материалов (>2 лет без обновлений).

---

## Что делать агенту при старте нового проекта (расширенный ритуал)

После стандартного ритуала старта (читать 00-context-anchor.md + CHECKPOINT_LATEST.md)
дополнительно:

1. Прочитать `CONTEXT/ROADMAP.md` и `PROJECT_STATUS.md`.
2. Определить: какие технологии, задачи, пробелы есть в проекте.
3. Сформировать список навыков (по формату выше).
4. Вывести владельцу: «Изучил проект. Вот навыки, которые могут помочь:»
5. После ответа владельца — установить выбранные навыки в `.kiro/steering/`
   или записать отказанные в `CONTEXT/SKILLS_LOG.md` (чтобы не предлагать повторно).

## Журнал навыков (SKILLS_LOG.md)

Каждый проект ведёт `CONTEXT/SKILLS_LOG.md`:

| Дата | Навык | Источник | Статус | Причина отказа |
|------|-------|----------|--------|----------------|
| ГГГГ-ММ-ДД | название | ссылка | установлен / отклонён / отложен | — |

Агент не предлагает уже отклонённые навыки повторно (если владелец сам не попросит).

---

## Правило для УП (_PM)

УП при синхронизации правил в новый проект обязан:
1. Скопировать этот файл (`skill-suggestions.md`) в `.kiro/steering/` нового проекта.
2. Создать шаблон `CONTEXT/SKILLS_LOG.md` (пустая таблица).
3. Напомнить агенту нового проекта: «Прочитай проект и предложи навыки по протоколу skill-suggestions.md».

## Маркетплейсы скилов и MCP (проверенные, бесплатные)

Агент при поиске навыков обязан проверять в первую очередь эти источники:

| Источник | URL | Что там есть |
|----------|-----|-------------|
| **skills.sage** | https://skills.sage | Маркетплейс скилов. Установка в 2 клика командой из сайта. Сортировка по количеству установок |
| **smithery.ai** | https://smithery.ai | Маркетплейс скилов + MCP с категориями (coding, research, design, analytics). Видно провайдера (Anthropic, GitHub, Microsoft) |
| **Anthropic Cookbook** | https://github.com/anthropics/anthropic-cookbook | Официальные скилы и паттерны от Anthropic |
| **MCP Official** | https://github.com/modelcontextprotocol/servers | Официальные MCP-серверы |
| **Awesome MCP** | https://github.com/punkpeye/awesome-mcp-servers | Самый полный каталог MCP |

**Как искать на smithery.ai:** открыть → категория → найти нужное → скопировать команду → вставить в терминал.
**Как искать на skills.sage:** найти скилл → скопировать команду установки → вставить в терминал → Enter.

**Правило безопасности при установке:**
- Смотреть провайдера: Anthropic, Microsoft, GitHub, OpenAI — безопасно
- Проверять что в SKILL.md нет `curl`, `export`, `rm -rf` команд
- Непопулярные скилы от анонимных авторов — проверять вручную перед установкой

## Экономия токенов при работе со скилами (важно!)

- Агент читает только `name` + `description` из YAML front-matter скила — **не весь файл**
- Это означает: можно установить десятки скилов без перегрузки контекста
- Агент сам выбирает нужный скилл по описанию задачи — без указания пользователем
- Принудительный вызов: написать `/название-скила` или упомянуть в промте
- Если скилл нужен редко — создать из MCP: «создай skill из этого MCP» → агент сделает лёгкий вариант
