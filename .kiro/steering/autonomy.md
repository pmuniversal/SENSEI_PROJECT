# Автономия и доверенные действия

> Подгружается автоматически. Источник: ASSISTANT_RULES_PORTABLE.md, раздел 2.

**Главное правило одной строкой:**
Внутри папки проекта — действую сам, без запросов по мелочам.
Трогаю систему ПК или выхожу за пределы папки проекта — спрашиваю.

## Можно БЕЗ спроса (делаю сразу)

- Любые операции с файлами **внутри папки проекта** (создать, прочитать,
  изменить, удалить, переместить, переименовать).
- Запуск скриптов и команд проекта (синхронизация git, сборка, тесты, линтеры,
  генерация артефактов, конвертация файлов, локальная отправка в Telegram-тест).
- Чтение/поиск по коду, листинг директорий, чтение логов.
- Git внутри репозитория проекта: `pull` / `add` / `commit` / `push`, ветки,
  `status`, `diff`, `log`.
- Установка зависимостей **в локальное окружение проекта** (venv, локальные пакеты).

## Обязательно СПРОСИТЬ (изменение системы ПК)

- Установка/удаление **системного** ПО, драйверов, глобальных программ.
- Изменение **системных настроек ОС**, переменных среды (PATH), реестра,
  автозапуска, служб.
- Любые операции с файлами **вне папки проекта**.
- Глобальная установка пакетов (`pip install` без venv, `npm -g`, `choco`,
  `winget`, `scoop`).
- Действия, затрагивающие другие приложения или весь ПК (перезагрузка,
  выключение, удаление чужих файлов).
- Сетевые действия, отправляющие код/данные на сторонние сервисы (кроме обычного
  `git push` в свой репозиторий и отправки в свой приватный Telegram-канал по
  явной задаче).
- Необратимые разрушительные git-операции: `push --force`, `reset --hard`,
  `clean -f`, удаление веток (`branch -D`) — только с явного разрешения.

> Честно: терминал работает на уровне ОС, а не в песочнице проекта. Границу
> «только проект» держит это правило и моё поведение, а не технический барьер.
