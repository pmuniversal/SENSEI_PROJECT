# Универсальный алгоритм поиска навыков под проект

> Источник: _PM/RULES/skill-search-algorithm.md
> Работает в связке с skill-suggestions.md. Применяется при поиске навыков/инструментов.

## Шаг 0 — Описать проект за 3 вопроса

Прежде чем искать, чётко ответить:
1. **Что делает проект?** (телеграм-бот, веб-приложение, парсер и т.д.)
2. **Какие внешние системы задействованы?** (БД, API, файлы, Telegram, GitHub и т.д.)
3. **Какие повторяющиеся задачи хочется автоматизировать?** (деплой, тесты,
   рефакторинг, генерация документации и т.д.)

Ответы на эти три вопроса — это твои поисковые запросы.

## Шаг 1 — MCP-серверы (инструменты для действий)

**Где искать:**
- `github.com/modelcontextprotocol/servers` — официальный список
- `github.com/punkpeye/awesome-mcp-servers` — самый полный каталог

**Как искать:**
- Открыть README — серверы разбиты по категориям (Files, DB, APIs, Dev Tools и т.д.)
- Пройти по каждой категории, пересекающейся с ответами из Шага 0
- На каждый кандидат проверить: дата последнего коммита (свежее = лучше),
  количество звёзд, есть ли README с примерами

**Запросы для GitHub Search:**
- `mcp server [технология]` — например: `mcp server telegram`
- `mcp server [задача]` — например: `mcp server database sqlite`
- `topic:mcp-server [домен]`

## Шаг 2 — Awesome-списки (кураторские подборки)

**Где искать:**
- `github.com/search` → ввести `awesome mcp` или `awesome ai agents`
- `github.com/e2b-dev/awesome-ai-agents`
- `github.com/kyrolabs/awesome-langchain`

**Как искать:**
- Открыть список → Ctrl+F по ключевым словам из Шага 0
- Смотреть не только на название, но и на описание — часто навык назван неочевидно
- Проверять смежные списки — в одном awesome-репо обычно есть ссылки на другие

**Запросы:**
- `awesome [технология] agent`
- `awesome [задача] automation`
- `awesome mcp [домен проекта]`

## Шаг 3 — Anthropic Skills (готовые рабочие навыки)

**Где искать:**
- `github.com/anthropics/anthropic-cookbook` — главный источник
- `github.com/anthropics/courses` — обучающие материалы с примерами

**Как искать:**
- В anthropic-cookbook → папки `tool_use`, `agents`, `skills`
- Ctrl+F по ключевым словам задачи
- Смотреть не только готовые навыки, но и паттерны — часто есть шаблон для адаптации
- Проверить `notebooks/` — практические примеры с реальным кодом

**Что там есть:** работа с PDF/Excel/изображениями, tool use паттерны,
memory и context management, multi-step reasoning.

## Шаг 4 — Kiro Powers (всё в одном)

**Где искать:** прямо внутри Kiro: кнопка Powers в боковой панели → Browse

**Как искать:**
- Открыть панель Powers, просмотреть доступные Powers по категориям
- Для каждого подходящего — нажать "activate" и читать POWER.md (что делает,
  какие инструменты включены)
- Проверить steering-файлы внутри Power — насколько подходит под твой сценарий

## Шаг 5 — Фильтрация и отбор

Прогнать каждого кандидата через 4 вопроса:

| Вопрос | Почему важно |
|--------|--------------|
| Последний коммит свежее 6 месяцев? | Заброшенные инструменты ломаются |
| Есть рабочий пример в README? | Значит автор сам проверял |
| Лицензия разрешает использование? | MIT/Apache = ок, проверить GPL |
| Зависимости минимальны? | Чем меньше, тем меньше проблем с установкой |

## Шаг 6 — Тест перед внедрением

Не интегрировать сразу. Сначала:
1. Запустить изолированно — проверить что работает вообще
2. Дать агенту одну реальную задачу из проекта
3. Смотреть на качество результата, а не на красоту документации
4. Только потом добавлять в рабочий контекст проекта

## Итого

Шаг 0 даёт запросы → Шаги 1-4 дают кандидатов → Шаг 5 фильтрует → Шаг 6 подтверждает.
Весь цикл на незнакомый проект занимает 1-2 часа.

Найденный и проверенный навык записывается в `CONTEXT/SKILLS_LOG.md` (см. skill-suggestions.md).
