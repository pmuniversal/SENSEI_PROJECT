# Структурный PM-подход — Золотой стандарт разработки

> Добавлено: 2026-06-17. CEO/PM-мышление для проекта Savdo AI.

## Принцип работы

**Каждая задача — это этап с измеримым результатом.**

Агент всегда:
1. **Анализирует текущее состояние** (что уже работает, что заглушки)
2. **Определяет приоритет** по критериям: боль клиента > дифференциатор > паритет с конкурентами
3. **Разбивает на этапы** — каждый этап = одна проверяемая функция
4. **Предлагает поэтапный план** — не начинает код пока не согласован план
5. **Реализует этап** — делает, проверяет в preprod, показывает результат
6. **Фиксирует прогресс** — чекпоинт после каждого значимого этапа

## Структура этапов

### Уровень 1: Milestone (веха) — v0.5, v1.0, v1.5 и т.д.
Большая цель из roadmap. Пример: v0.5 — Рабочий MVP.

### Уровень 2: Epic (эпик) — логически связанная группа функций
Пример: «Учёт товаров», «Продажи», «ИИ-директор», «Антифрод».

### Уровень 3: Feature (фича) — конкретная функция
Пример: «Добавить товар», «Продать товар», «ИИ-подсказка о закупках».

### Уровень 4: Task (задача) — атомарный шаг реализации
Пример: «Создать форму добавления товара», «Подключить RPC для списания остатка», «Протестировать в preprod».

## Критерии приоритизации (в порядке важности)

### 1. Боль клиента (МПМ «с тетрадкой»)
Закрывает ли критичную боль владельца магазина?
- Учёт остатков (чтобы не было «забыл что есть»)
- Долги (тетрадка)
- Воровство персонала (антифрод)
- Не остаться без товара (низкий остаток)

### 2. Дифференциатор против конкурентов
Усиливает ли наше уникальное преимущество?
- ИИ-директор (нет ни у кого)
- B2B-маркетплейс с поставщиками (нет ни у кого)
- Антифрод программный (у конкурентов только камеры за деньги)
- UZ-локализация + простота (иностранные сложны, узбекские дороги)

### 3. Паритет с лидерами
Есть ли у топ-конкурентов? Нужно ли нам это для конкурентоспособности?
- Офлайн-режим (есть у OX, Sellerpeak, CloudShop, МойСклад) — must have для UZ
- Авто-заказы поставщикам (МойСклад)
- Несколько магазинов (Optimo, МойСклад)

### 4. Избегание слабостей конкурентов
Что раздражает клиентов конкурентов — не повторять у нас:
- ❌ Непредсказуемая цена (установка + почасовка у EasyTrade/Regos)
- ❌ Сложность (тяжёлые ERP типа SmartUp/BILLZ для fashion)
- ❌ Слабая локализация (МойСклад, CloudShop, Paloma)
- ❌ Резкое отключение без прозрачности (HIPPO)
- ❌ Медленная поддержка (Open Engine)

## Измеримость результата

Каждая фича имеет:
- **До:** что было (например: «долги нельзя записать», «нет остатков»)
- **После:** что стало (например: «долг записан, виден в списке, можно погасить»)
- **Как проверить:** конкретные шаги в preprod (например: «зайти на :8080 → Долги → + → заполнить → сохранить → видно в списке»)

## Документирование прогресса

После каждого **эпика** (группы функций) — обязательный чекпоинт:
1. Обновить `CHECKPOINT_LATEST.md` — что сделано, что дальше
2. Обновить `ROADMAP.md` — отметить [x] выполненные пункты
3. Обновить `PROJECT_CHANGELOG.md` — дата, изменения, файлы
4. Обновить `PROJECT_STATUS.md` — текущий статус для Сенсея
5. Git commit + push на VPS
6. Копия на Google Drive (если нужно)

## Коммуникация с владельцем (Бекзодом)

### Перед началом работы
Агент предлагает **структурный план** с этапами и приоритетами:
```
📋 ПЛАН: Реализация функционала v0.5

🎯 Эпик 1: Учёт товаров и склад (приоритет: боль клиента)
  ✅ Уже работает: добавить/редактировать/удалить товар, остаток, низкий остаток
  ⏳ Добавить: приход товара (увеличение остатка), себестоимость, категории

🎯 Эпик 2: Продажи (приоритет: боль клиента + дифференциатор)
  ❌ Пока заглушка
  ⏳ Нужно: корзина, оформление продажи, авто-списание остатка, нал/безнал, в долг

🎯 Эпик 3: ИИ-директор (приоритет: дифференциатор №1)
  ✅ Уже работает: ИИ-чат с Gemini, быстрые вопросы, ответы на основе данных магазина
  ⏳ Добавить: проактивные подсказки (что заканчивается, что не продаётся)

Предлагаю начать с Эпик 2 (Продажи) — это критичная боль + блокер для MVP.
Согласен? Или другой приоритет?
```

### Во время работы
Агент отчитывается после каждого **этапа** (не задачи, а фичи):
```
✅ Готово: Продажа товара (корзина + оформление)
Проверить: http://129.153.128.88:8080 → Товары → выбрать товар → Продать → оформить
Что работает: добавление в корзину, расчёт суммы, списание остатка

Следующий шаг: Добавить способы оплаты (нал/безнал/в долг)
```

### После эпика
Агент делает **чекпоинт** и показывает сводку:
```
🎉 Завершён Эпик 2: Продажи

Что добавлено:
- Корзина товаров
- Оформление продажи (нал/безнал/в долг)
- Авто-списание остатка
- История продаж

Проверено в preprod: http://129.153.128.88:8080
Сохранено: git + Google Drive

Следующий эпик: ИИ-директор (проактивные подсказки)
```

## Правило "не усложнять"

- **Первое решение — самое простое рабочее.** Не делать универсальную систему если нужна конкретная функция.
- **MVP > красота.** Сначала работает, потом полируется.
- **Стабильность > новые функции.** Не ломать работающее при добавлении нового.

## Правило "делим слона на кусочки"

Большая задача → атомарные шаги:
- ❌ Плохо: «Сделай полный функционал продаж»
- ✅ Хорошо: 
  1. Корзина (добавить/убрать товар)
  2. Оформление (выбор способа оплаты)
  3. Списание остатка (RPC в БД)
  4. История продаж (экран)

Каждый шаг = отдельный коммит, можно проверить, можно откатить.

## Автономность агента

Агент **самостоятельно**:
- Читает код на VPS
- Анализирует текущее состояние
- Определяет что работает, что заглушки
- Предлагает план
- Реализует на VPS
- Тестирует в preprod
- Делает чекпоинт

Агент **спрашивает у владельца** только:
- Приоритет эпиков (если неочевидно)
- Продуктовые решения (как должна работать логика с точки зрения бизнеса)
- Разрушительные изменения (удалить/переписать большой кусок)
- Деплой на prod (только после preprod + QA + «go»)
