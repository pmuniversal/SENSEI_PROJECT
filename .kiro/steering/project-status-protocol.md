# Протокол ведения PROJECT_STATUS.md

> ⭐ ВЫСШИЙ ПРИОРИТЕТ. Этот файл читает Сенсей-бот для мониторинга всех проектов.
> Ключевое слово владельца "ИСП" (Изучи Состояние всех Проектов) → Сенсей читает
> PROJECT_STATUS.md каждого проекта и формирует сводный отчёт.

## Главное правило

**PROJECT_STATUS.md обновляется на КАЖДОМ чекпоинте — без исключений.**
Если файл не обновлён — чекпоинт не считается завершённым.

## Обязательная структура файла

```
# PROJECT STATUS — [Название проекта]

> ⭐ Этот файл читает Сенсей. Обновляется на каждом чекпоинте.

## Продукт / Направление
[Одно предложение: что это и для кого]

## Текущий этап
[Название этапа по ROADMAP.md]

## Статус: [🟢 АКТИВЕН / 🟡 ПАУЗА / 🔴 ЗАБЛОКИРОВАН / ⚪ СТАРТ]

## Что сделано (последнее)
- [дата] [что сделано — одна строка]
- [дата] [что сделано — одна строка]

## В работе сейчас
[Что делается прямо сейчас, одна строка]

## Следующий шаг
[Конкретное следующее действие]

## Метрики / KPI (если есть)
[Ключевые цифры: DAU, MRR, задеплоено, % готовности и т.д.]

## Блокеры / Риски
[Что мешает или может помешать — или "нет"]

## Обновлено: YYYY-MM-DD
```

## Правила заполнения

1. **Кратко и по делу** — Сенсей читает ВСЕ проекты сразу, нужна ясность.
2. **Статус одним эмодзи:** 🟢 активен / 🟡 пауза / 🔴 заблокирован / ⚪ старт.
3. **"Что сделано" — максимум 3-5 строк**, только последнее.
4. **"Следующий шаг" — одно конкретное действие**, не список.
5. **Дата обновления — обязательна** (YYYY-MM-DD).

## Когда обновлять

- После каждого значимого изменения (задача выполнена, этап завершён).
- При смене статуса (пауза, блокер, возобновление).
- При изменении следующего шага.
- Минимум раз в сессию, даже если "ничего не изменилось" — написать текущее состояние.

## Как Сенсей использует этот файл

При команде владельца **"ИСП"** (Изучи Состояние всех Проектов) Сенсей:
1. Читает PROJECT_STATUS.md каждого проекта.
2. Формирует сводную таблицу: проект → статус → текущий этап → следующий шаг.
3. Выделяет блокеры и риски.
4. Даёт рекомендацию: на чём сосредоточиться сегодня.

## Запрещено

- Оставлять пустые поля или плейсхолдеры типа "(заполнить)".
- Пропускать обновление на чекпоинте.
- Писать длинные абзацы — только конкретные факты.
