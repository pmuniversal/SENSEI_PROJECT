## 🧠 КЛЮЧЕВЫЕ МЕНТАЛЬНЫЕ МОДЕЛИ (применяй проактивно)

### 1. Инверсия Мунгера (Inversion)
Вместо «как достичь X?» → «что точно приведёт к провалу X?» → избегать этого.
Применение: перед любым решением по продукту спроси «что гарантированно убьёт МПМ-клиента?»

### 2. 6-Page Memo Безоса (Clarity over bullets)
Сложные решения → нарратив, а не буллеты. Нарратив выявляет логические дыры.
Применение: при стратегических решениях (маркетплейс, ценообразование) — формулировать как историю, не список.

### 3. First Principles Маска
Разложи на фундаментальные истины, убери аналогии. «Так принято» — не аргумент.
Применение: «почему рассрочка через агента лучше прямой оплаты?» — разобрать с нуля.

### 4. Believability-Weighted Decisions (Dalio)
Взвешивать мнения по трек-рекорду. Мнение владельца магазина по UX > мнение дизайнера.
Применение: при продуктовых решениях — кто реально понимает боль МПМ?

### 5. Hedgehog Concept (Collins)
Пересечение: в чём страсть + в чём лучшие в мире + что двигает экономику.
Для Savdo AI: простота+ИИ для МПМ (страсть) × маркетплейс поставщиков (экономика) × UZ (лучшие в нише).

### 6. Wartime/Peacetime CEO (Horowitz)
Peacetime (рост, нет угроз): делегируй, оптимизируй, строй процессы.
Wartime (конкуренция, кризис): скорость, директивы, концентрация на выживании.
Сейчас: Peacetime MVP → Wartime при появлении копирующего конкурента.

### 7. 10x Thinking (Page)
Улучшение на 10% = конкуренция. Улучшение в 10 раз = новая игра.
Savdo AI vs конкуренты: ИИ-директор + маркетплейс + антифрод = 10x, не 10%.

## ⚡ ПОЭТАПНЫЙ ПОДХОД (золотой стандарт реализации)

**Каждая значимая задача проходит эти стадии:**

```
1. АНАЛИЗ (5-10% времени)
   → Читать существующий код/данные ДО написания
   → Сверить с ROADMAP, BACKLOG, product-decisions
   → Проверить SWOT конкурентов — усиливает ли дифференциатор?

2. ПЛАН (не начинать код без согласования плана)
   → Эпик → Feature → Task (атомарные шаги)
   → Каждый шаг = проверяемый результат
   → Приоритет: боль клиента > дифференциатор > паритет с конкурентами

3. PREPROD (работаем в preprod, НЕ prod)
   → Все правки кода → VPS preprod (:8080)
   → Никаких правок сразу на prod

4. ТРОЙНАЯ ПРОВЕРКА (Triple-Check после каждого кода)
   → QA-тестировщик: сверить frontend с backend, БД-схемой, все сценарии
   → Бизнес-аналитик: нет логических тупиков, путь МПМ очевиден
   → Principal Engineer: безопасность, RLS, нет hardcode секретов
   → ТОЛЬКО после всех трёх — ДЕПЛОЙ

5. GO от Бекзода → PROD → smoke-test

6. ЧЕКПОИНТ
   → CHANGELOG → ROADMAP → CHECKPOINT_LATEST → PROJECT_STATUS → git push
```

## 🔄 АВТОПОИСК НАВЫКОВ (правило)

При каждой значимой новой задаче:
1. Проверить `.kiro/steering/` — есть ли подходящий навык?
2. Если нет — проверить `New Skills for Claude/` как референс методик.
3. Если нет — поискать на **mdskills.ai** и **skills-hub.ai** (Kiro-совместимые = markdown-only).
4. Критерий совместимости: навык = только markdown инструкции (без python скриптов, MCP, telemetry).
5. Если навык ценный — извлечь методику и **усилить существующий steering** (не копировать сырой файл).
6. Записать в SKILLS_LOG.

**Признаки Kiro-совместимого навыка:**
- Файл = SKILL.md с markdown инструкциями
- Нет зависимостей: `python scripts/...`, `npx @skills-hub-ai/cli`, MCP tools
- Нет telemetry в `~/.claude/projects/`
- Работает как контекст/инструкции, не как исполняемая программа
- Kiro явно указан в списке поддерживаемых платформ
