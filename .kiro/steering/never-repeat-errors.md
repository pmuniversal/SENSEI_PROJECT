# Протокол учёта ошибок — Never Repeat

> Обнаружил ошибку — зафиксируй. Повторить ту же ошибку — запрещено.

## Главное правило

**Любая обнаруженная ошибка записывается в `CONTEXT/ERRORS_LOG.md` немедленно.**
Перед каждым действием — проверить лог. Если похожая ошибка уже есть — применить
известное решение, не наступать на те же грабли.

## Что считается ошибкой (записывать всё)

- Баг в коде (логика, синтаксис, runtime, интеграция)
- Неверное предположение об архитектуре или данных
- Ошибка в тексте (неверный факт, противоречие, неточность)
- Неправильная команда терминала или git-операция
- Неверное понимание требований владельца
- Сбой деплоя, конфига, переменных окружения
- Любое действие, которое пришлось откатывать

## Формат записи в ERRORS_LOG.md

```
## [YYYY-MM-DD] [категория] Краткое название ошибки

**Что произошло:** одна строка — суть ошибки
**Где:** файл/модуль/команда/контекст
**Причина:** почему возникло
**Решение:** что сделали чтобы исправить
**Как не повторить:** конкретное правило на будущее
**Статус:** исправлено / в работе / принято к сведению
```

### Категории
- `[КОД]` — ошибки в коде
- `[ТЕКСТ]` — ошибки в документах, фактах, формулировках
- `[ДЕПЛОЙ]` — проблемы с инфраструктурой, сервером, конфигами
- `[ЛОГИКА]` — неверные предположения, архитектурные ошибки
- `[КОМАНДА]` — неверно понята задача владельца
- `[ПРОЧЕЕ]` — всё остальное

## Когда проверять лог

1. **Перед началом новой задачи** — просмотреть ERRORS_LOG.md на похожие ошибки.
2. **При повторяющемся паттерне** — если делаешь что-то похожее на прошлый раз.
3. **При старте сессии** — прочитать лог вместе с CHECKPOINT_LATEST.md.

## Правила ведения лога

- Записи не удаляются — только добавляются сверху (новые выше старых).
- Не нужно описывать подробно — достаточно чтобы понять суть и не повторить.
- Если ошибка системная (встречается везде) — поднять в _PM для добавления в эталон.
- Лог ведётся в `CONTEXT/ERRORS_LOG.md` каждого проекта.

## Запрещено

- Повторять ошибку, которая уже есть в ERRORS_LOG.md.
- Не записывать ошибку после исправления ("и так запомню").
- Удалять или редактировать старые записи.
