# SENSEI — техника (как устроено и где работает)

> Подгружается автоматически в каждом чате. Техническое состояние «как есть».

## Стек

- **Язык:** Python 3.11
- **Бот:** pyTelegramBotAPI (telebot)
- **AI:** OpenAI API, модель `gpt-4.1-mini`; голос → текст через Whisper API
- **БД:** SQLite (таблицы: `memory`, `tasks`, `reminders`)
- **Документы:** python-docx (сохранение диалогов/транскриптов в DOCX)
- **Аудио:** pydub (+ ffmpeg — на сервере ставится отдельно)
- **Секреты:** python-dotenv (читает `.env`)

## Архитектура кода (после рефакторинга 02–03.06.2026)

Точка входа снаружи НЕ менялась: Supervisor по-прежнему запускает `/app/main.py`,
а это теперь тонкий «переходник» на `bot/main.py`.

```
CODE/
├── main.py                 # переходник → bot/main.py
├── .env / .env.example     # токены (.env НЕ в git)
├── .gitignore / requirements.txt
├── STRUCTURE.md            # карта проекта (подробно)
├── MIGRATION_CHECKLIST.md  # перенос на сервер + откат
├── bot/
│   ├── main.py             # точка входа: хэндлеры + фоновые потоки + polling
│   ├── config.py           # токены/пути из .env, fail-fast если токена нет
│   ├── handlers/           # text.py, voice.py, commands.py
│   ├── services/           # ai.py, memory.py, tasks.py, reminders.py, backup.py
│   ├── database/db.py      # подключение + init таблиц (схема неизменна)
│   └── utils/              # folders.py, docx.py
└── data/                   # memory.db, backups/, docs/, voice/
```

## Важные технические факты

- **`SENSEI_DATA_DIR=/app`** на сервере — рабочая база осталась в `/app/memory.db`
  (данные НЕ переносили в `data/`, чтобы не потерять). Помнить при изменениях путей.
- **Системные env-переменные имеют приоритет** над значениями из `.env`.
- Если токена нет — бот падает с понятной ошибкой (не молча).
- В логах успешного старта есть строка **`AI OS STARTED`**.
- Бэкап БД: фоновый поток раз в час. Напоминания: фоновый поток каждые 30 сек.
- Память контекста: последние **15 сообщений** загружаются как контекст диалога.
- **Голосовые ОТВЕТЫ бота ОТМЕНЕНЫ** (решение владельца) — бот отвечает только текстом.
  Расшифровка голоса В текст остаётся.

## Команды бота (текущие)

- `/task <текст>` — создать задачу
- `/tasks` — список активных задач (баг с перехватом веткой /task — ИСПРАВЛЕН)
- `/remind YYYY-MM-DD HH:MM | текст` — напоминание

## Типовые серверные команды (Docker)

```
docker ps                          # список контейнеров
docker restart simplerllm          # перезапуск
docker logs simplerllm             # логи (искать "AI OS STARTED")
docker exec -it simplerllm bash    # войти в контейнер
```
Полное руководство: `VPS/docker_commands.md`, `VPS/deployment.md`, `VPS/server_info.md`

## Технические запреты/ограничения (договорённости)

- НЕ использовать LangChain на раннем этапе.
- НЕ использовать Vector DB до стабильной памяти.
- Ядро простое и контролируемое. Новые функции — отдельными модулями, не ломая ядро.
- НЕ подключаться к серверу напрямую (нет SSH-доступа у ИИ) и НЕ принимать
  пароли/ключи доступа в чат. Перенос — через пошаговые команды владельцу.
