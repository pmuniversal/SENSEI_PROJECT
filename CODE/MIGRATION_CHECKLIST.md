# Чек-лист переноса на сервер и проверки

Этот документ — пошаговая инструкция, как безопасно перенести новую версию
бота на сервер (VPS) и проверить, что всё работает. Написано простыми словами.

> ВАЖНО: пока ты не выполнил эти шаги на сервере, твой старый бот продолжает
> работать как прежде. Новую версию мы включаем только в самом конце.

---

## ЧАСТЬ 0. Перед началом — заменить старые токены (ВАЖНО про безопасность)

Старые токены были записаны прямо в коде, поэтому их нужно считать
«засвеченными» и заменить на новые.

### Шаг 0.1 — Новый токен Telegram
1. Открой в Telegram бота **@BotFather**.
2. Отправь команду `/mybots`.
3. Выбери своего бота.
4. Нажми **API Token** → **Revoke current token** (отозвать текущий токен).
5. BotFather выдаст **новый токен** — скопируй его.

### Шаг 0.2 — Новый ключ OpenAI
1. Зайди на сайт **platform.openai.com**.
2. Открой раздел **API keys**.
3. Удали старый ключ (тот, что начинается на `sk-proj-x6a7...`).
4. Нажми **Create new secret key** — скопируй **новый ключ**.

### Шаг 0.3 — Вписать новые токены в файл .env
1. Открой файл `.env` в папке проекта.
2. Замени значения на новые:
   ```
   TELEGRAM_TOKEN=сюда_новый_токен_от_BotFather
   OPENAI_API_KEY=сюда_новый_ключ_от_OpenAI
   ```
3. Сохрани файл.

> Токены вписываются ТОЛЬКО в файл .env. Нигде в коде их больше нет.

## ЧАСТЬ 1. Резервная копия (страховка)

Зайди в контейнер на сервере:
```
docker exec -it simplerllm bash
```

Сделай копии на всякий случай:
```
cp /app/main.py /app/main.py.bak
cp /app/memory.db /app/memory.db.bak
```

Дополнительно скачай копию `memory.db` к себе на компьютер через Portainer.

---

## ЧАСТЬ 2. Загрузка новых файлов

Загрузи на сервер в папку `/app` новую структуру:
- папку `bot/` целиком
- новый `main.py` (переходник)
- файлы `.env`, `.env.example`, `.gitignore`, `requirements.txt`
- (по желанию) `STRUCTURE.md`, `MIGRATION_CHECKLIST.md`

Это можно сделать через Portainer (загрузка файлов) или командой `docker cp`.

---

## ЧАСТЬ 3. Перенос базы данных в папку data/

Цель — положить базу и папки в новую папку `data/`, не потеряв данные.

```
mkdir -p /app/data/backups /app/data/docs /app/data/voice
mv /app/memory.db /app/data/memory.db
mv /app/docs/*    /app/data/docs/    2>/dev/null || true
mv /app/backups/* /app/data/backups/ 2>/dev/null || true
```

### Более безопасный вариант (если переносить базу страшно)
Можно НЕ переносить базу, а сказать боту использовать старое место.
Для этого задай переменную окружения:
```
SENSEI_DATA_DIR=/app
```
Тогда бот продолжит читать базу из `/app/memory.db`, как раньше.

## ЧАСТЬ 4. Установка новой библиотеки

```
pip install python-dotenv
```
(или `pip install -r requirements.txt`)

Если пропустить — бот при запуске сам подскажет, что не хватает библиотеки.

---

## ЧАСТЬ 5. Перезапуск

```
docker stop simplerllm
docker start simplerllm
docker logs simplerllm
```

В логах должна появиться строка: **AI OS STARTED**

> Настройки Supervisor менять НЕ нужно — он запускает тот же /app/main.py.

---

## ЧАСТЬ 6. Проверка (сделай в Telegram)

| № | Что сделать | Что должно произойти |
|---|-------------|----------------------|
| 1 | Написать любой текст | Бот отвечает осмысленно |
| 2 | Написать ещё пару сообщений | Помнит, о чём говорили |
| 3 | `/task Купить молоко` | «✅ Задача добавлена» |
| 4 | `/tasks` | Показывает СПИСОК задач (не создаёт новую!) |
| 5 | `/remind 2099-01-01 10:00 \| Тест` | «⏰ Напоминание сохранено» |
| 6 | `/remind кривой ввод` | Подсказка формата |
| 7 | Отправить голосовое | Текст голоса + ответ + Word-файл |
| 8 | Посмотреть `docker logs simplerllm` | Есть «AI OS STARTED», нет ошибок |

Если все пункты прошли — миграция успешна. 🎉

---

## ЧАСТЬ 7. Если что-то сломалось — откат

```
docker stop simplerllm
cp /app/main.py.bak /app/main.py
cp /app/memory.db.bak /app/memory.db
docker start simplerllm
```

Старый `main.py.bak` самодостаточен (содержит всё необходимое),
поэтому откат полностью возвращает рабочую версию.

После успешной проверки копии `.bak` можно удалить (но лучше хранить пару недель).
