# Структура проекта Sensei

Этот документ — карта проекта. Он показывает, где что лежит и за что отвечает.

## Дерево файлов

```
CODE/
├── .env                      # СЕКРЕТНЫЕ токены (НЕ попадает в git)
├── .env.example              # Шаблон-образец без настоящих ключей
├── .gitignore                # Что git должен игнорировать
├── requirements.txt          # Список нужных библиотек Python
├── STRUCTURE.md              # Этот файл — карта проекта
├── MIGRATION_CHECKLIST.md    # Пошаговый чек-лист переноса на сервер
├── main.py                   # Переходник: запускает bot/main.py
│
├── bot/                      # ВЕСЬ код бота
│   ├── __init__.py
│   ├── main.py               # Точка входа: собирает и запускает бота
│   ├── config.py             # Загрузчик токенов и путей (читает .env)
│   │
│   ├── handlers/             # Приём сообщений из Telegram
│   │   ├── __init__.py
│   │   ├── text.py           # Обычные текстовые сообщения → AI
│   │   ├── voice.py          # Голос → распознавание → AI
│   │   └── commands.py       # Команды /task, /tasks, /remind
│   │
│   ├── services/             # Логика работы (что бот умеет)
│   │   ├── __init__.py
│   │   ├── ai.py             # Обращение к OpenAI + системный промпт
│   │   ├── memory.py         # Память (сохранение/загрузка диалогов)
│   │   ├── tasks.py          # Задачи (добавить/показать)
│   │   ├── reminders.py      # Напоминания + фоновая проверка
│   │   └── backup.py         # Авто-бэкап базы раз в час
│   │
│   ├── database/             # Работа с базой данных
│   │   ├── __init__.py
│   │   └── db.py             # Подключение + создание таблиц
│   │
│   └── utils/                # Вспомогательные инструменты
│       ├── __init__.py
│       ├── folders.py        # Создание рабочих папок
│       └── docx.py           # Сохранение диалогов в Word (DOCX)
│
├── data/                     # ВСЕ данные (НЕ попадает в git)
│   ├── memory.db             # База данных
│   ├── backups/              # Копии базы (раз в час)
│   ├── docs/                 # Сгенерированные Word-документы
│   └── voice/                # Временные голосовые файлы
│
└── backups/
    └── main.py.bak           # Резервная копия старого main.py (точка отката)
```

## Принцип устройства (слои)

Код разделён на слои. Каждый слой обращается только к тем, что «ниже» его:

```
main.py (переходник)
   └── bot/main.py (точка входа)
         ├── handlers/  (приём сообщений)
         │     └── services/  (логика)
         │           └── database/  (база данных)
         ├── config.py  (токены и пути — им пользуются все)
         └── utils/     (вспомогательное)
```

## Как добавлять новые функции в будущем

Структура сделана так, чтобы расширять её БЕЗ переделки существующего кода:

- **Новая команда** → добавить обработчик в `bot/handlers/commands.py`
  или создать новый файл в `bot/handlers/`.
- **Новая логика** (например, финансы или цели) → создать новый файл
  в `bot/services/` (например, `services/finance.py`).
- **Новая таблица в базе** → добавить `CREATE TABLE IF NOT EXISTS ...`
  в `bot/database/db.py` (старые данные не пострадают).
- **Новая настройка** → добавить константу в `bot/config.py`.

Существующие файлы при этом трогать не нужно — это и есть «модульность».

## Будущий этап: проактивный «супер-мозг»

Когда дойдём до фазы, где Сенсей сам пишет тебе в Telegram, отслеживает цели
по сферам жизни и спрашивает о прогрессе — это добавится новыми файлами:

- `bot/services/proactive.py` — когда и о чём писать самому
- `bot/services/goals.py` — цели по сферам жизни
- `bot/services/scheduler.py` — планировщик проактивных действий

Ядро при этом меняться не будет.
