# Google Sheets Интеграция — Полная Инструкция

## 📋 Обзор

Теперь Сенсей ведёт две Google Таблицы:
1. **"Кредиты"** — все условия, остатки, ставки, статусы
2. **"История"** — каждый платёж с датой, суммой, комментарием

Финансовый парсер автоматически пишет в таблицы при упоминании платежей в диалогах.

---

## ✅ Шаг 1: Создать Google Таблицу

### 1.1 Перейдите на Google Sheets
- Откройте https://sheets.google.com
- Нажмите **"+ Создать"** → **"Новая таблица"**
- Переименуйте в **"SENSEI_FINANCE"** (или как вам нравится)

### 1.2 Скопируйте ID таблицы
- В адресной строке URL выглядит так:
  ```
  https://docs.google.com/spreadsheets/d/ЗДЕСЬ_ID_ТАБЛИЦЫ/edit
  ```
- Скопируйте часть между `/d/` и `/edit` — это ваш **GOOGLE_SHEETS_ID**
- Пример: `1a2b3c4d5e6f7g8h9i0j1k2l3m4n5o6p`

---

## ✅ Шаг 2: Настроить Google Cloud

### 2.1 Создать проект в Google Cloud Console
1. Откройте https://console.cloud.google.com
2. В левом меню нажмите на название проекта → **"Новый проект"**
3. Назовите его **"SENSEI-BOT"** → нажмите **"Создать"**
4. Подождите 1-2 минуты, пока проект активируется

### 2.2 Включить Google Sheets API
1. В Google Cloud Console откройте **"APIs & Services"** → **"Library"**
2. Найдите **"Google Sheets API"**
3. Нажмите на неё → **"Включить"**
4. Подождите несколько секунд

### 2.3 Создать Service Account
1. Вернитесь в **"APIs & Services"** → выберите **"Credentials"**
2. Нажмите **"+ Create Credentials"** → **"Service Account"**
3. Заполните:
   - Service account name: **sensei-bot-sheets**
   - Description: **Sensei Financial Tracker**
   - Нажмите **"Create and Continue"**

### 2.4 Создать ключ (JSON)
1. На странице Service Account нажмите на созданный аккаунт
2. Перейдите в закладку **"Keys"**
3. Нажмите **"Add Key"** → **"Create new key"** → выберите **JSON**
4. Автоматически скачается файл **`xxxxx-key.json`** — сохраните его!

### 2.5 Получить Email Service Account
1. Откройте скачанный JSON-файл в любом текстовом редакторе
2. Найдите строку `"client_email": "sensei-bot-sheets@..."` 
3. Скопируйте этот email (вам понадобится на следующем шаге)

---

## ✅ Шаг 3: Дать Доступ Service Account к Таблице

### 3.1 Поделиться таблицей с Service Account
1. Откройте вашу таблицу **"SENSEI_FINANCE"** на Google Sheets
2. Нажмите кнопку **"Поделиться"** (верхний правый угол)
3. В поле для email вставьте email Service Account (из шага 2.5)
4. Выберите разрешение **"Редактор"** (Editor)
5. Нажмите **"Поделиться"**

---

## ✅ Шаг 4: Загрузить Файл Учётных Данных на Сервер

### 4.1 На вашем компьютере (локально)
1. Откройте скачанный JSON-файл (из шага 2.4)
2. Переименуйте в **`google_credentials.json`**

### 4.2 На сервере (Portainer)
1. Откройте **Portainer** → выберите контейнер **telegrambot**
2. Перейдите в **"Files"** (или в консоль)
3. Перейдите в директорию `/app`
4. Загрузите файл **`google_credentials.json`** туда
5. Убедитесь, что путь: `/app/google_credentials.json`

> **Альтернатива**: если Portainer не имеет интерфейса загрузки, используйте SSH:
> ```bash
> scp -r /путь/к/google_credentials.json user@ваш_сервер:/app/
> ```

---

## ✅ Шаг 5: Добавить Переменные Окружения

### 5.1 Редактировать `.env` на сервере
1. В Portainer откройте консоль контейнера **telegrambot**
2. Отредактируйте файл `.env`:
   ```bash
   nano /app/.env
   ```

3. Добавьте две строки:
   ```env
   GOOGLE_SHEETS_ID=1a2b3c4d5e6f7g8h9i0j1k2l3m4n5o6p
   GOOGLE_CREDENTIALS_PATH=/app/google_credentials.json
   ```
   Где:
   - `GOOGLE_SHEETS_ID` — это ID из шага 1.2
   - `GOOGLE_CREDENTIALS_PATH` — путь к JSON-файлу

4. Сохраните (Ctrl+O, Enter, Ctrl+X)

### 5.2 Альтернатива: редактировать локально
1. На вашем компьютере откройте `.env` файл
2. Добавьте те же две строки
3. Синхронизируйте на сервер через Git или SFTP

## ✅ Шаг 6: Установить Зависимости и Перезагрузить

### 6.1 В консоли контейнера (Portainer)


### 6.2 Перезагрузить бота

## 🎯 Использование

### Автоматическое логирование платежей
При упоминании платежей в диалогах финансовый парсер автоматически:
1. Добавляет запись в лист **"История"** с датой, временем и суммой
2. Обновляет остаток в листе **"Кредиты"**

### Из Python кода
```python
from bot.services.sheets import log_payment, log_early_repayment

# Логировать обычный платёж
log_payment(
    credit_name="Анорбанк №1",
    amount=2500000,
    comment="Оплачено через мобильный банк",
    new_balance=15045179
)

# Логировать досрочное погашение
log_early_repayment(
    credit_name="AVO карта",
    amount=5000000,
    comment="Мониторинг просрочки"
)
```

## 🔍 Возможные Проблемы и Решения

### ❌ Ошибка: "Файл google_credentials.json не найден"
**Решение:**
1. Проверьте, что файл загружен на сервер в `/app/`
2. Проверьте имя файла (должно быть точно `google_credentials.json`)
3. Проверьте переменную `GOOGLE_CREDENTIALS_PATH` в `.env`

### ❌ Ошибка: "403 Forbidden"
**Решение:**
1. Проверьте, что таблица **поделена** с email Service Account
2. Проверьте, что email скопирован правильно (без пробелов)
3. Убедитесь, что разрешение выставлено на **"Редактор"**

### ❌ Ошибка: "Spreadsheet not found"
**Решение:**
1. Проверьте, что ID таблицы скопирован правильно
2. Проверьте переменную `GOOGLE_SHEETS_ID` в `.env`
3. Убедитесь, что таблица еще существует

### ❌ Сервис недоступен — функция отключена
**Решение:**
1. Проверьте, что зависимости установлены:
   ```bash
   pip list | grep google
   ```
2. Если отсутствуют, установите:
   ```bash
   pip install google-auth-oauthlib google-auth-httplib2 google-api-python-client
   ```
3. Перезагрузите бота
