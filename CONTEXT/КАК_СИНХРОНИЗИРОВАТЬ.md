# 🔄 Как синхронизировать проект (ноут + Google Диск + GitHub)

> Цель: чтобы во всех трёх местах всегда была одна и та же свежая версия,
> и любой ИИ-агент мог продолжить без потери контекста.

---

## Три места, где живут файлы

1. **Ноут** — папка `G:\Мой диск\KIRO_PROJECTS\Sensei\SENSEI_PROJECT` (где ты работаешь).
2. **Google Диск** — **синхронизируется САМ**, потому что папка внутри `G:\Мой диск\...`.
   Ничего нажимать не нужно.
3. **GitHub (приватный)** — `github.com/pmuniversal/SENSEI_PROJECT`. Сюда нужно
   «отправить» изменения. Это и делает кнопка ниже.

---

## Способ 1 (самый простой) — двойной клик

1. Открой папку проекта.
2. Двойной клик по файлу **`СОХРАНИТЬ.bat`**.
3. Откроется чёрное окно, оно само:
   - добавит изменения,
   - сделает точку сохранения (commit),
   - отправит в GitHub (push).
4. Когда увидишь «ГОТОВО» — всё в трёх местах синхронно. Окно закроется само.

> Если напишет «Не удалось отправить в GitHub» — проверь интернет и что ты вошёл
> в GitHub Desktop (там должен быть твой аккаунт `pmuniversal`).

---

## Способ 2 — через ИИ (Kiro)

Просто скажи: **«Сохрани контекст»** или **«Синхронизируй проект»**.
Если включён хук «Синхронизировать контекст», это произойдёт автоматически,
когда я закончу работу над задачей.

---

## Способ 3 — вручную через GitHub Desktop

1. Открой GitHub Desktop.
2. Слева внизу впиши в **Summary** что менял (например «обновил чекпоинт»).
3. Нажми **Commit to main**.
4. Сверху нажми **Push origin**.

---

## Что НЕ уходит в GitHub (и это правильно)

Защищено через `.gitignore` — остаётся только на ноуте и Google Диске:
- `.env` и `CODE/.env` — твои токены (Telegram, OpenAI);
- `CODE/data/` — база, голосовые, локальные бэкапы;
- `*.bak` — резервные копии.

Если случайно увидишь `.env` в списке изменений GitHub Desktop — НЕ коммить,
сразу скажи мне, поправим.

---

## Правило одной строкой

**Закончил работу → запусти `СОХРАНИТЬ.bat` → всё в трёх местах синхронно.**
