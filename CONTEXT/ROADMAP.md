# SENSEI ROADMAP (главный маршрут развития)

> Это жёсткий маршрут, по которому идём по порядку. Каждый этап — отдельный
> мини-проект: сначала план, потом аккуратная реализация, потом проверка.
> Принцип: Стабильность > Красота > Новые функции. Не ломать рабочее.
> Владелец — не программист, поэтому каждый шаг объясняется простым языком.

# ЭТАПЫ (идём строго по порядку)

## ЭТАП 1 — Защита памяти: копия в Telegram-канал ⭐ ПЕРВЫМ
**Зачем:** память (memory.db) — самое ценное. Сейчас копии лежат на том же сервере
(риск: умрёт сервер — пропадёт всё). Делаем копию «наружу».
- [ ] Раз в час бот шлёт свежий `memory.db` в приватный TG-канал (бот там уже админ)
- [ ] Команда «сделай копию сейчас» — отправка по требованию
- [ ] Дополнительно: текстовый/markdown экспорт памяти (профиль + история) —
      файл, который можно скормить ЛЮБОМУ ИИ, и он «узнает» пользователя
- [ ] Также периодически слать в канал архив кода (для восстановления бота)

## ЭТАП 2 — Голосовые сообщения (рус + узб) → Word/txt + резюме встреч
**Зачем:** скидывать голосовые и записи совещаний, получать готовый текст.
- [ ] Доустановить ffmpeg (программа-«переводчик» аудиоформатов)
- [ ] Качественная транскрипция Whisper (русский + узбекский)
- [ ] Выдача результата файлом Word и txt
- [ ] Режим «резюме встречи»: главные мысли, решения, задачи из записи
- [ ] Поддержка длинных записей (совещания)

## ЭТАП 3 — Умный ввод: человеческий язык вместо команд
**Зачем:** надоело писать /task и /remind в строгом формате.
- [ ] Бот сам понимает: это разговор, задача или напоминание
- [ ] «в сфере ютуба задача создать канал, дедлайн 10 июня» → задача в сферу YouTube + дедлайн
- [ ] «напомни 10 числа открыть канал» → напоминание без формата
- [ ] Привязка задач к 4 сферам автоматически
- [ ] Команда «что у меня по ютубу?» → задачи только этой сферы
- [ ] Старые команды (/task, /tasks, /remind) остаются как запасной вариант

## ЭТАП 4 — Второй слой бэкапа: Google Диск
**Зачем:** ещё одно безопасное место для копий (кроме TG-канала).
- [ ] Настроить доступ к Google Диску (ключ доступа — проведём пошагово)
- [ ] Память + код дублируются на Google Диск
- [ ] (взято из BACKLOG, делаем после Этапа 1)

## ЭТАП 5 — Личный финансист
**Зачем:** бот знает финансовое состояние, контролирует траты, советует.
- [ ] Хранение: доходы, расходы, долги, кредиты, источники дохода
- [ ] Ввод человеческим языком: «потратил 50к налички на еду»,
      «за 300к с карты купил жене одежду»
- [ ] Категории и сферы трат
- [ ] Баланс и финансовые отчёты (неделя/месяц)
- [ ] Бот предупреждает: куда нежелательно тратить, куда можно
- [ ] Учёт кредитов/долгов и напоминания о платежах
- [ ] Инвестиционный дневник (позже)

## ЭТАП 6 — Проактивность («супер-мозг сам пишет первым»)
**Зачем:** бот держит фокус, сам напоминает о целях, спрашивает о прогрессе.
- [ ] Бот сам инициирует сообщения (не ждёт, пока напишут)
- [ ] Утренний брифинг: задачи и напоминания на день по всем сферам
- [ ] Вечерний разбор: что сделано, что перенести
- [ ] Спрашивает о прогрессе по целям («как продвигается канал на ютубе?»)
- [ ] Не даёт распыляться и прыгать между сферами (удержание фокуса)
- [ ] Дедлайны: напоминает заранее

## ЭТАП 7 — Персональный профиль и долгая память
**Зачем:** чтобы бот «знал тебя» и ничего не забывал.
- [ ] Профиль: цели по сферам, ценности, привычки, решения
- [ ] Поиск по прошлым диалогам
- [ ] Умное сжатие длинной истории (чтобы помнил больше 15 сообщений)
- [ ] Анализ прогресса и паттернов поведения

## ЭТАП 8 — Анализ документов
- [ ] PDF, DOCX, Excel/CSV
- [ ] Суммаризация и ответы на вопросы по документу

## ЭТАП 9 — Трекеры (личное развитие)

## ЭТАП 12 — Исламское мировоззрение и духовные напоминания

## ЭТАП 13 — Интеграции (БЕЗ голосовых ответов)
> ВАЖНО: голосовые ОТВЕТЫ бота ОТМЕНЕНЫ по решению владельца — бот отвечает ТОЛЬКО текстом.
> (Расшифровка голоса В текст остаётся — это Этап 2.)

- [ ] **Google Calendar** — задачи и дедлайны автоматически попадают в календарь
- [ ] **Рабочий Gmail** (приоритет — это рабочий инструмент, не вишенка):
      - Сенсей отслеживает входящую почту от клиентов из США
      - Когда приходит ответ клиента — бот РЕАГИРУЕТ: уведомляет владельца в Telegram,
        кратко излагает суть письма, подсказывает/помогает с ответом
- [ ] **Trello** (витрина задач для удобства):
      - Сенсей создаёт карточку задачи в нужной колонке
      - Двигает карточку по статусам, закрывает завершённые
      - Владелец видит на доске Trello все свои задачи (быт/работа/бизнес/финансы)
      - Доска отражает контроль выполнения по 4 сферам
- [ ] ПРИМЕЧАНИЕ: интеграции требуют «ключей доступа» (Google, Trello). Проведём пошагово.

## ПРАВИЛА ДВИЖЕНИЯ ПО ROADMAP
- Идём по этапам по порядку (1 → 14), не перепрыгиваем без причины.
- Каждый этап = отдельный мини-проект (план → реализация → проверка).
- После каждого этапа — запись в PROJECT_CHANGELOG.md (чекпоинт).
- Стабильность важнее новых функций. Перед изменениями — бэкап.
- Все объяснения — простым языком, пошагово.
