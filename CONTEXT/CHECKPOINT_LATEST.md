## СЛЕДУЮЩИЕ ШАГИ

### 🔴 АКТИВНАЯ ЗАДАЧА: GitHub Actions автодеплой (Вариант 2)

**Цель:** при каждом `git push` сервер сам подтягивает изменения и перезапускает бота. Никаких ручных команд в Portainer.

**Где остановились:** выбрали вариант, начали настройку SSH-ключа.

**Следующий шаг — выполнить на хост-Ubuntu VPS (не внутри контейнера!):**

```bash
# Шаг 1 — сгенерировать SSH-ключ
ssh-keygen -t ed25519 -C "github-actions" -f ~/.ssh/github_actions -N ""

# Шаг 2 — добавить публичный ключ в authorized_keys
cat ~/.ssh/github_actions.pub >> ~/.ssh/authorized_keys

# Шаг 3 — показать публичный ключ (отправить мне для проверки)
cat ~/.ssh/github_actions.pub

# Шаг 4 — показать ПРИВАТНЫЙ ключ (добавить в GitHub Secrets — никуда не отправлять!)
cat ~/.ssh/github_actions
```

После этого:
- Приватный ключ → GitHub репо → Settings → Secrets → `SSH_PRIVATE_KEY`
- IP сервера → GitHub Secrets → `SSH_HOST` = `129.153.128.88`
- Создать файл `.github/workflows/deploy.yml`

**Доступ к хост-Ubuntu:** через PuTTY напрямую на `129.153.128.88` ИЛИ Portainer → Host Console (не контейнер!)

## КАК ОБНОВЛЯТЬ ФАЙЛЫ НА СЕРВЕРЕ (важно!)

❌ НЕ использовать curl — кеширует GitHub
✅ Использовать python3 urllib:
```bash
python3 -c "
import urllib.request
url = 'https://raw.githubusercontent.com/pmuniversal/SENSEI_PROJECT/main/CODE/bot/services/ФАЙЛ.py'
req = urllib.request.Request(url, headers={'Cache-Control': 'no-cache', 'Pragma': 'no-cache'})
content = urllib.request.urlopen(req).read().decode()
open('/app/bot/services/ФАЙЛ.py', 'w').write(content)
print('OK:', len(content))
"
```
